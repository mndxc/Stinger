# Getting Started with Stinger Development

This guide will help you initialize your development environment, configure the IDE properly, and build the Stinger firmware for the ESP32-S3. Due to some quirks with ESP-IDF and the VS Code IDE, **please follow these steps carefully** to avoid CMake and IntelliSense errors.

## 📋 1. Prerequisites (macOS)

You'll need a few system tools. If you don't have Homebrew installed, get it from [brew.sh](https://brew.sh).

```bash
brew install cmake ninja dfu-util python3 git
```

## 🛠️ 2. ESP-IDF v6.0 Setup (via VS Code Extension)

The Stinger project requires **ESP-IDF v6.0**. It is highly recommended to manage this via the official VS Code Extension for automated tooling support.

1. **Install the Extension:** 
   Search for and install the official **Espressif IDF** extension in your VS Code extension manager.
2. **Run the Setup Wizard:** 
   Open the Command Palette (`Cmd+Shift+P`) and type `ESP-IDF: Configure ESP-IDF extension`.
3. **Install the Framework:**
   Choose **Express Setup**, select **v6.0** from the version dropdown, and allow the wizard to automatically download the framework and ESP32-S3 toolchains.

---

## ⚠️ 3. CRITICAL: IDE & Workspace Configuration

The most common cause of build failures (`Configure failed`, `include could not find requested file`) or false-positive code errors in this project stems from incorrect folder structures.

1. **Open the correct Workspace Folder:** 
   Do **NOT** open the root `Stinger` repository directory as your primary workspace folder in VS Code if you are doing firmware development. You **MUST** open the `firmware/stinger` directory as the root folder. 

2. **Verify Settings (`.vscode/settings.json`):**
   The project contains a pre-configured `.vscode/settings.json` located inside `firmware/stinger/`. It explicitly tells the IDE and CMake where the IDF tools are:
   - Sets `"cmake.environment": { "IDF_PATH": "..." }` so CMake doesn't fail on including the ESP-IDF boilerplate.
   - Sets Clangd arguments and disables conflicting IntelliSense parsing to prevent false `-mlongcalls` errors in IntelliSense.

3. **Verify C/C++ Properties (`.vscode/c_cpp_properties.json`):**
   This configures IntelliSense to use `compile_commands.json` generated by the build system.

*Note: If you change machines or ESP-IDF versions, you may need to update the absolute paths inside the `.vscode/settings.json` and `.vscode/c_cpp_properties.json` variables to match your local environment.*

---

## 🏗️ 4. Build & Flash Firmware

Using the extension toolbar at the bottom of VS Code:

1. **Set Target:** Click the device icon or run `ESP-IDF: Set Espressif device target` and select `esp32s3`.
2. **Build:** Click the ⚙️ (Build) icon or run `ESP-IDF: Build your project`.
3. **Flash & Monitor:** Click the ⚡️ (Flash) and 📺 (Monitor) icons to upload your code and view serial output.

*(You can also use the `ESP-IDF: Open ESP-IDF Terminal` command, where `idf.py build` will work automatically).*

---

## 🤖 5. Automated Pin Configuration

The Stinger project uses a single-source-of-truth for GPIO configurations.

**Do NOT manually edit `main/pin_map.h` or the layout tables in `SPEC.md`.**

If you need to change a pin assignment:
1. Edit `firmware/stinger/master_pins.json`.
2. Save your changes.
3. The build system automatically triggers `tools/generate_pins.py` which will regenerate `pin_map.h` and explicitly patch your changes into the `SPEC.md` root specification file.

---

## 📂 Project Structure

- `firmware/stinger/` — Main ESP-IDF project. **Open this directory in VS Code for firmware dev.**
  - `master_pins.json`: Hardware GPIO definitions (Single source of truth).
  - `main/board_config.h`: Master constants (Audio/USB/Display).
  - `tools/generate_pins.py`: Python code generation script.
- `hardware/` — KiCad design files (Brain and Plate boards).
- `SPEC.md` — Technical Specification.
